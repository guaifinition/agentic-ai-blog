## 一、愿景与组织方式

我和公司的其他同事最初创办这家公司时，并没有预设最终要赚多少钱，也没有以进入资本市场或上市为目标。

最初的几十位成员也没有这种想法；如果以财富或资本化为首要目标，他们可能不会加入。总体而言，我们怀着对世界的善意推进这项工作，并认为它对人类具有价值，其意义超越金钱本身。

随着人工智能潜在利益不断扩大，外部诱惑也随之增加，但这是另一个层面的问题。公司的出发点、愿景以及至今保持的方向，并不是按照商业利益最大化的原则设定的。这一点非常关键。

大约二十年前，我在管理领域最推崇的人是通用电气前 CEO 杰克·韦尔奇。现在回看，他的很多观点可能已经不再适用，但其中有一点仍然成立：对一家公司而言，最重要的是愿景。

管理一家大型公司，不能只依靠规章制度，更重要的是愿景。愿景不是挂在墙上的标语，而是公司实际如何行动、如何运行。其核心不在于如何表述，而在于如何实践。

我们如何管理并组织如此多的人？严格地说，公司并没有传统意义上的严密组织体系，而是主要依靠愿景驱动。

这种方式既有优势，也有缺点，未来需要扬长避短，但它构成了公司的显著特征。我们并不主要依靠 KPI 或考核组织工作，而是依靠共同愿景。

这个愿景甚至没有形成正式文本。它存在于我们做事的方法以及对待世界的态度之中。公司成员对愿景的具体理解可能不同，个人追求也会存在差异，但在大的方向上是一致的。

其共同基础仍是以善意面对世界，并尝试做一些真正有价值的事情。公司正是依靠这种共识组织起来的。

后续内容将围绕这一愿景展开。这个愿景并非事后编造，我们确实如此思考，也确实如此行动；否则，公司的许多选择就无法得到一致解释。

## 二、开源的动机与商业逻辑

我们之所以坚持开源，首先是因为愿景本身要求如此。没有这种愿景，就很难把团队长期组织起来。

其他公司也可能开源，但不同公司的动机并不相同。以智谱为例，我的感受是其开源更像外部压力下的选择；对我们而言，开源则是初衷的一部分。

在开源问题上，我们一开始就考虑得比较清楚：第一，开源符合愿景；第二，我们认为，要在商业上真正把人工智能做成，开源本身是有利的。

这听起来可能违反传统商业直觉。历史上，开源往往与软件商业化存在冲突，因为传统软件市场总体规模有限。一家软件公司的年市场规模可能只有几十亿美元，一旦完全开源，原有收入空间可能会显著缩小。

但人工智能不同。它所影响的经济活动规模极大，最终可能占据全球 GDP 中相当高的比例。面对这样一个规模的产业，任何单一主体都不可能独占全部利益，必须与社会和产业链中的其他参与者分享。

这与传统软件开源不同，因为人工智能的市场规模远大于单一软件品类。如果试图独占全部利益，最终会被历史抛弃。这首先是一种对客观规律和产业演进的判断。

闭源并不意味着能够独占市场，这在理论上也不符合客观事实。试图实现垄断，必然会遭遇来自竞争、政策、产业和社会层面的多重阻力。

因此，不能简单沿用传统商业逻辑。企业需要建立一种机制，主动限制自身获取利益的范围；只有保持克制，才更有可能把事情做成。

如果希望由我们推动这项事业取得成功，首先必须保持克制。不能设想全球或中国 GDP 中某个很大的比例最终都归属于一家公司；越以此为目标，越难实现长期成功。

因此，我们从一开始就认为需要克制。越克制，越可能把事情做成。这既是商业考量，也是宏观战略判断。

至少在我的判断中，这一逻辑符合直觉。我们并没有显著资源优势：资金不比别人充足，团队也并非天然优于其他公司。

公司成立之初，资金有限、算力有限，也缺乏知名度和号召力，团队成员只是一群普通人。

我们确实只是一群普通人。我们更认同“一群普通人共同完成不平凡的事情”，而不是“一群天才完成不平凡的事情”。这种叙事与公司的克制和愿景是一脉相承的。

开源是否会与商业化冲突？在人工智能领域，如果缺乏克制，就很难建立可持续的长期模式。

开源是克制的一部分，而公司的克制还体现在许多其他方面。对我们而言，这并不是临时权衡，而是基本原则。

越克制，越可能把事情做成；至少到目前为止，这一逻辑能够解释公司的发展。我们没有特殊武器，起点较低、资源有限，团队也只是由一群普通人组成。我本人也只是普通大学毕业生，并非出身于最顶尖院校。

## 三、合理利润与 API 定价

克制也是愿景的一部分。人工智能的潜在利益足够大，只要真正把事情做成，即使只获得其中很小的一部分，也已经足够。因此，现阶段没有必要优先考虑应当获取哪一部分利益、通过什么方式获取。

只要获得其中很小的一部分利益，就已经足够。因此，我们主张只获取合理利润，而不是追求利润最大化。具体到 API 定价，我们认为，从市场采购一批设备后约十个月收回设备成本，是可以接受的商业回报。

在当前条件下，考虑技术风险、前期投入和运营风险，服务器在财务上可以按照三年或五年折旧，但商业定价上，我们认为约十个月收回设备成本已经足够。这是现阶段 API 定价的基本逻辑。V3.2 Flash 等服务均大致遵循这一标准。

这并不是利润最大化定价。如果以利润最大化为目标，价格应当更高。因为在当前价格区间，用户需求弹性较低，即使价格提高一半甚至一倍，Token 消耗量也可能不会显著下降。

如果价格提高一倍，而需求基本不变，总收入也会接近增加一倍。

此前某款模型〔原始 ASR 名称无法确认〕上线时，我们担心需求过大，因此最初定价较高，团队内部对此并不满意。后来价格降至原来的约四分之一，团队反而非常高兴。

这更能反映公司的真实想法：我们希望模型对人有用，而不是让公司赚取最多利润；在获得合理利润的前提下，应尽量让更多人能够负担。模型降价时，公司内部出现积极反馈，正是这种共同价值取向的体现。

我们投入大量精力优化模型，目的就是使其效果更好、成本更低，让社会能够广泛使用。这是团队的动力、愿景和内部共识。

这一点可能与部分竞争者不同。对许多商业公司而言，降价会直接压低年度经常性收入，因此未必值得庆祝；而我们的团队更重视使用门槛和社会价值。

从公司内部看，十个月收回设备成本已经具有足够的商业吸引力；从社会角度看，这一价格也能形成企业与用户之间的双赢。

有人指出，十个月回本仍意味着较高收益。确实，价格和模型优化都还有进一步空间，因此未来仍可能继续降价。

但按照十个月回本定价，需要具备相应的系统和成本优化能力。我的判断是，部分大型公司在这一点上的实际成本可能高出数倍，仍有大量优化工作需要完成。

没有继续立即降价，主要是因为当前需求对价格已经不敏感。价格进一步下降，需求增量可能很小，因为现有价格已经不会成为多数用户停止使用的主要原因。

因此，进一步降价既不会显著增加公司收入，也不会明显提高社会效用。定价的出发点不是收入或利润最大化，而是在合理利润、普遍可负担和长期战略之间取得平衡。从短期看，提高价格可能增加收入；从长期看，克制本身是一种策略。

## 四、克制是一种战略

对我而言，克制是一种战略：在某些方面主动舍弃短期利益，以换取更重要的长期条件。开源也是如此，可以视为一种主动让利。

这种让利首先会增强组织内部的认同感。员工能够从中获得成就感，团队凝聚力也会增强。同时，开源和低价对社会、同行及普通用户都有直接价值。因此，从长期看，克制能够提高我们最终实现 AGI 的概率。

我并不怀疑 AGI 会具有巨大的商业价值。在这一前提下，我优先考虑的不是如何多占一点份额，而是如何提高成功概率。

这种克制也体现在用户和产品策略上。某年春节期间，用户数量突然大幅增加，但我们没有把留存、变现或争夺流量视为第一目标。

我们没有通过大规模投入争抢用户或短期收入，但仍然尽力把现有用户服务好。公司并不以成为下一个超级应用、下一个字节跳动或下一个腾讯为目标。

这并非能力不足，而是主动选择。拥有大量用户并不意味着必须立即建立封闭商业生态；不追求覆盖全部价值链，也是克制的一部分。

从商业角度看，投入大量资金与头部平台争夺用户也可能是一种可行策略，但我们选择不在这一阶段展开正面竞争，因为后续 AGI 机会远大于当前流量机会。

即使当前机会规模不小，相对于未来仍可能只是阶段性收益。现在回看，前一阶段没有把主要资源投入 C 端竞争，可能是正确选择。

即使当时拥有更多资金并把 C 端业务迅速做大，也未必能获得与投入相称的长期价值。我的基本判断是，后续 AGI 机会远大于当前应用层机会。

现阶段甚至不必过早确定未来在 AGI 产业中的具体位置或商业模式。只要总体机会足够大，商业路径自然会出现。对于阶段性机会，我们会以较低成本承接，但不会因此中断技术主线。

因此，前一阶段的 C 端日活并非最高优先级，但我们仍以较低成本维持用户服务，因为这些用户未来可能具有价值。即使现阶段主要体现为成本支出，也没有必要主动放弃。

对于能够以较低额外成本获得的业务机会，我们会自然承接。API 等 B 端业务也可能形成可观的年度经常性收入；如果需求持续扩大、算力采购能够跟上，年收入达到数亿美元并非没有可能。

如果相关收入进一步增长，公司现金流可能转正，并覆盖研发及其他运营费用。但这仍然不是公司的第一优先级。

C 端和 B 端业务需要做，也应当做好，但它们不是公司的终极目标。更大的机会仍在后续 AGI 技术突破。

或者说，我们公司大部分人不认为这是一个非常重要的事情，不认为这是一个跟 AGI 比较起来同等重要的问题。

## 五、开源与商业化的兼容性

关于开源，我们会继续坚持，最强模型也可能开源，因为目前看不到闭源的必然优势。

即使模型完全开源，真正高质量地部署和运行仍有很高门槛。第三方不仅要使模型正确运行，还要把推理成本降到足够低。

原理公开并不意味着每家公司都能够达到同等部署成本。这里涉及大量工程细节，也取决于企业是否愿意并有能力组织足够人力。

一些公司可能并不擅长控制部署成本，因为会受到管理结构和物理系统的多重约束。这也是创业公司的相对优势：规模过小则资源不足，规模过大又可能难以高效组织。

因此，我们当前的公司规模处在一个相对适宜的区间：规模更小会力量不足，规模更大则可能产生新的组织问题。

至于开源后的商业服务，我们不会收取过高费用，仍会按照约十个月回收设备成本的原则定价。按这一价格提供服务，已经足以形成很强的成本竞争力。

按照十个月回本的价格，独立部署的第三方很难获得利润，因为通常无法达到同等成本水平。

因此，在合理利润前提下，开源不会显著影响收入；只有在追求极高超额利润时，开源才会明显压缩利润空间。

原文以“六倍利润”描述十个月回本所对应的长期收益倍数。该表述并非严格财务术语，但核心意思是：只追求合理收益时，开源与商业付费可以长期并存。

如果试图获取数十倍乃至百倍的超额利润，第三方即使部署成本更高，也可能以更低价格进入市场。

这一模式可以长期持续。在公司的愿景下，开源是长期战略；它既体现克制，也通过短期让利换取长期利益。

这一策略使我们能够把更多机会和资源保留给技术主线，提高实现 AGI 的概率，并保持更从容的节奏。

由于目标高度聚焦，我们通常不需要长期加班。相较之下，若同时追求过多目标，组织复杂度会显著上升。

外界可能认为，选择基础研究并解决最困难的问题是一条高难度路线。但我们在其他方向主动舍弃了大量工作，因此组织更加聚焦，整体负担反而较轻。

因此，我的判断是，开源可以长期持续；在合理收益范围内，开源与商业付费不存在根本冲突。

这一收益倍数表面上较高，但在当前 AI 效率和设备生命周期条件下，可以被视为合理水平。未来可能下降到更低倍数，但仍会保留可持续利润。

即使只看 API 业务，我也没有看到开源与商业服务之间存在不可调和的冲突；不过，API 收入本身并不是最具吸引力的长期目标。

我们不仅不担心第三方部署模型，还希望开源社区能够顺利部署。

我们愿意尽可能提供帮助。真正担心的是第三方由于细节处理不当，导致效果下降或成本过高，而不是其与我们竞争。

这里是没有冲突的。

过去有人担心：模型开源后，拥有流量优势的平台可以直接部署同一模型，从而分流我们的 C 端用户。

实际运行中并未出现这种根本冲突。

开源模型与自有服务使用的是同一版本。我们不会开源较弱模型，再在自有产品中使用更强版本。

过去一段时间的 C 端实践也没有显示开源与自有服务之间存在根本冲突。

## 六、长期目标：AGI

下面还有就是，公司的长期愿景，我认为我们目标应该是 AGI。每个人对 AI 定义不一定一样，但是不妨碍我们把 AGI 当做我们的目标。

从技术路线上来看，其实 AGI 这条路线图是比较清晰的。跟目前的这一代 AI 技术，如果说你能够把一个问题描述得很清楚，给它完整的上下文和指令，它已经超过人类了。但这里有个定义，有个前提是：你给它完整的上下文，你给它完整的指令。

然而，“完整上下文”这一前提在现实中很难满足。例如，一次会议背后包含参与者数十年的知识和经验，而当前 AI 并不具备这些长期上下文。它能够在有限上下文中表现得优于人类，但仍难以进入真实组织环境。

因此，当前 AI 仍无法普遍替代人类员工，关键缺口是持续学习。新员工可以用两个月熟悉公司环境、工作内容和协作关系，之后便能独立开展工作。

他就能做很多事情。他能够听懂你说的话，比如说，你说，叫那个小王来，他就知道小王是谁。但如果 AI 的话，因为这个上下文，它前面没有这两个月的学习。你跟它说，叫小王过来，你得告诉它小王是谁、什么职务、他在哪里、要怎么去找他、要找他的时候注意什么。你得给 AI 所有的上下文。那么在这种情况下，AI 是能做的，但是你不可能给它所有的上下文，也不现实。

因此，AI 目前还不能像员工一样进入组织。如果 AI 具备持续学习能力，能够像员工一样在公司学习两个月并形成长期经验，其可替代范围将大幅扩大。下一步仍缺少的是“学会学习”。

AI 的发展，我们可以理解成它是一个阶梯。去年走的阶梯是 CoT，就是思维链。因为我们发现，通过思维链的方式，可以让智能达到一个更高的水平。通过它自己思考，可以让这个上限，可以让这个 AI 能做更多的事情。

之后又出现了 Agent 这一台阶。通过 Agent 机制，模型能够处理更长、更复杂的任务流程，能力范围和智能上限进一步提高。

为什么是阶梯呢？因为后面的每一步都是基于前面的基础上的。Agent 要用到 CoT，然后 CoT 也要用到前面的阶梯，前面的阶梯就是语言模型，所以它并没有一步是白走的。

所以，AI 的发展，智能的走向是有迹可循的。说今年走的这个阶梯是 Agent，但是 Agent 这个阶梯，它也是会走完的。就是它把所有可能解决的问题都解决完之后，但是它还是不能替代你的员工，但是它已经到它能力的上限了。

就好像 CoT 一样，CoT 到它的上限之后，它已经超过最顶尖的人类了，在做奥数题、写程序上面已经超过最顶尖的人类了。但是它还是停在那个地方，它那个技术并没有能够到达 AGI。

所以你看，AI 这个智能的走向，它是有迹可循的。然后在 Agent 之后，我们觉得应该要解决的问题是持续学习，就是怎么让模型可以持续地学习，而不是说你要给它一个很强的训练，它应该能够像人一样做一个比较长时间的持续学习。

持续学习与长期完成任务本质上高度相关。站在 Agent 阶段，我们能够看到的下一个主要瓶颈就是持续学习；这一问题相对明确。

就是卡在前面的这个障碍，你必须要跨过去，而且一定是有办法能跨过去的，但需要时间。持续学习之后，可能我们就会来到一个奇点。这个奇点就是，当这个模型能够持续学习之后，它已经能够做人类能做的所有事情了。

它就能够自己开发自己的版本，能够自己再研究，然后开发自己的下一个版本，开发更前的人工智能模型。所以它就会到一个奇点，能够实现自己的迭代。

所谓“奇点”未必是真正的瞬时突变，更可能是持续时间较长的渐进过程，只是习惯上仍使用“奇点”这一概念。

早期预测常把这一阶段描述为明确奇点，但我认为它更可能是连续演进。完成这一阶段之后，才更适合进入具身智能。

这是我们的推测，这是我们觉得这个时间表应该是：先解决学习去学习，然后再到那个智能的奇点，能自我迭代的奇点，然后才是具身智能。到具身智能之后，它就走进现实世界，可以给你做家务，可以给你养老。

我们觉得这是一个比较理想的路线图，但每个人的观点不一样，没有对错之分。只是我们觉得，这个路线图是最轻松的。

这个路线图是因为每一步你要做新的都很少。这个路线图我们可…6 tokens truncated…不是我们不想 Scaling，是我们没有那么多的算力去做这个 Scaling。

我们没有触摸到这个上限在哪里。我们训练这么大的模型，并不是因为我认为这么大的模型就够了，而是我刚好有这么多资源。我是按照我的资源来算，我能够接受、能够训练的模型是多大，是这样算出来的，并不是这个模型就够了。

目前来看，模型收益还是非常明显的。我们还没有机会碰到这个 Scaling 的墙，这离我们还很远。硅谷在说 Scaling 到头的时候，那是对硅谷来说；对中国人来讲，我们离那个还很远，我们没有 Scaling 到那个程度。

这个 Scaling 包括数据的 Scaling、模型规模的 Scaling，然后训练成本。我们离探索这个 Scaling 的上限还比较远，就是没有那么多算力。但我们也会不遗余力地去推进这个 Scaling 的上限。

融资后，公司将获得更多算力，并可能训练更大模型。设备越早到位越好；如果能在半年内完成有效采购是理想状态，但现实上很难做到。

下一代模型的核心能力，我认为它必须要有持续学习的能力，它才能叫下一代模型。在那之前，我们能做的就是成本，然后效果做得更好，速度做得更快。但是要有大突破，它应该是具备持续学习的。

## 十六、计算效率、合理利润与组织管理

我们特别重视模型计算效率，但并非所有公司都会把效率置于同等优先级。对部分以商业收入为首要目标的公司而言，降低推理成本未必直接转化为更高短期收益。

因此，部分创业公司并不会把极致低成本作为核心目标，因为较高服务成本和定价可能更符合其短期商业利益。

相反，这是我们愿景的一部分。或者说，我们的愿景，我们的同学在乎这个成本，因为我们的同学都是普通人，他知道用这个都是要花钱的。他能够有这个同理心：别人要花钱来用，那么别人如果便宜一点，别人能够接受的程度会更高。

团队中很多成员仍希望持续降低成本。但若仅从传统商业角度出发，低成本未必会被列为第一优先级。

从成本或者从商业角度讲，这不是第一优先级。

对于不管是创业公司来讲，还是大公司来讲，这个服务成本不高。但是我认为我们希望它是比较轻的，我希望它是一个负担得起的，特别是在中国算力紧缺的这样的背景下，是负担得起的，能在国产卡上用的。

我认为低成本首先是一个结果。我们的模型确实一直在模型架构上往一个更低成本的方向走，这跟我们的愿景有关系。我们还有很多在算法上的方法，成本还可以往下走。

成本往下走还有一个原因是，成本越低，我就越能训练更大的模型，我就越能承担起更大的模型。在同样算力上，在算力有限的情况下，如果我的计算效率更高，我就能够承担起更大的模型。

对大公司来讲，它不一定会这么考虑。对大公司来讲，资源是可以加的，可以通过加资源来解决。但是我们会优先考虑成本效率。

数据的范围很广，可以认为构成模型能力的重要一半。进一步从商业逻辑看，假设 AI 最终占全球 GDP 的 20%，任何公司若试图独占其中 5%，都可能被只要求 1% 收益的竞争者击败。

如果说我的目标是，我要占 AI、全人类 GDP 的百分之五，理论上算这个账还是成立的。你看 OpenAI，算这个账好像是能算得过来的，理论上是没问题的。但是他有个问题，他会被另外一个愿意只占百分之一的人打败。

因为另外一个人说，我做得这么好，但是我只要拿全球 GDP 的百分之一就可以了，那么就会把他打败。这时候如果又出来另外一个人，说我只要百分之零点一就可以了，那么又会把前面的人给打败。

从宏观上来讲，不管这百分之几是从哪里拿，彼此之间没有区别，都是一样。拿得多的人会被拿得少的人打败。甚至你还不用真的拿得多，愿景如果是拿得多的话，你就会被愿景是拿得少的人给打败。

其实大家都没有拿到钱，然后只是一个愿景。你愿景是拿得多，你就先输了，你就会面临着更大的困难。这个世界就是这样。

如果一家公司从一开始就以垄断全球市场为目标，就会遭遇大量挑战者。未来美国公司还会面对中国公司的竞争，因为中国企业可能愿意以更低收益提供同等服务。

在中国，也会有人愿意拿得更少一点。但最后这里会有个平衡，因为你拿得太少之后，公司商业逻辑就不成立，就活不下去了。所以你拿得太少了，你活不下去；你拿太多了，你会被拿得少的人打败。

所以对我们来讲，我们并不是利润要拿最多的钱，或者说算收益最大化的定价，而是只赚一个合理的收益。这是一个解释。

我是相信这个事的，我并不是去为这件事找理由，因为没必要找理由。我本来就这么做的，我这么做必然是有理由的。这个理由可能不是非常惯常，但是我认为公司本来就不惯常。

我们公司的管理其实是两条线：一条线是从上到下，一条是从下到上。从下而上，就是每个人自己想做什么，自己做，没有人管他，没有 KPI。

自上而下的管理主要用于正式集体项目。例如发布 V4 时，需要统一分工，每个人承担明确任务。我们把这类工作称为“正式任务”。

原则上，正式任务不应占据员工全部时间，最好不超过一半。其余时间不作统一安排，研究人员可以根据自己的判断探索重要问题，不设置前置要求。

只要公司能够支持，公司算力能够支持他做，或者说他不需要算力，他需要做得很少，那么他不用来协调。所以我们现在是这样的一个组织方式。

有些人觉得我们是从上而下的，有些人觉得我们是从下而上的，我认为两个都对。我的一个标准是，正式最好不要超过一半。

公司通常不鼓励长期加班。第一，研究需要相对宽松的环境；如果持续施加强压力，研究人员很难依靠兴趣进行长期思考和探索。这是研究文化的基本要求。

第二个是，我们非常聚焦。我们非常聚焦，就意味着我们要做的事情很少。那我就没那么多事情要做，我就不需要加班。

这与前面的克制是一脉相承的。因为我克制，所以很多时候我就不做了。那么我要做的事情少了，每个人分到的工作就少了。你看到我们的产品很多都不完善的，我们也没有去补它。

这也是我们的一种文化。

# 提问环节

> 问答者身份为原文后期标注，可能存在偏差。问题中的寒暄、重复确认和无实质内容已删除；技术名词无法确认处保留原识别并标注存疑。

## 1. 持续学习何时会带来突破？需要哪些创新？

**问：** 持续学习大约何时可能取得突破？实现持续学习还需要哪些架构、算法和其他关键要素？

**答：** 这个问题仍需要研究，全世界都在研究。投资人当前看到最多的是 Agent，但研究人员更关注“学习”本身以及如何解决学习问题。

持续学习不是一项单一技术，而是一个问题集合，可能需要多种技术共同解决。AGI 也不是由单一模型或单一算法构成，而是模型、算法、系统工程和其他能力的组合。当前有很多研究方向，但尚未形成公认的完整解法。

## 2. 开源生态、人才供给与产业结构

**问：** 开源之后，行业中有多少团队能够复现模型成果？未来生态发展需要哪些人才、算力和行业模型？中国算力短板需要多长时间解决？硬件进步与模型效率提升是否会共同降低算力压力？

**答：** 每个行业发展初期都会出现人才短缺，但这种短缺通常是阶段性的。互联网早期的网站开发、服务端工程都曾经缺人，后来都在两三年内形成了大规模人才供给。AI 人才短缺也会逐步缓解，而且已经在明显改善。企业培养相关人才的速度并不慢，因此人才不会成为长期结构性短缺。

国内做基础模型的公司目前偏多，导致资源分散和重复投入。美国基础模型研发相对集中，而国内很多公司做相似工作，每家公司能获得的资源更少。长期看，这一格局一定会收敛。当行业利润回归合理水平后，不会有那么多公司继续独立投入基座模型。中国最终可能保留三四家主要参与者，已经足以形成充分竞争。

大模型公司不可能拿走产业中的大部分利润。模型之间长期差异主要在时间和成本，成本控制得好就多赚一些，控制得差就少赚一些，不会长期存在不可持续的暴利。

我们愿意扶持更多生态伙伴，也不存在根本利益冲突，但公司精力有限，能够实际投入多少还取决于人力和优先级。

## 3. 真实数据、合成数据与超越人类

**问：** 如果模型最终能够自我学习和自我迭代，后续主要依赖真实数据还是仿真、合成数据？如果依赖人类已有真实数据，是否会限制模型超越人类？

**答：** 模型能够在一定范围内超越人类。AlphaGo 曾经走出人类此前未见的棋步，说明系统可以基于已有规则和知识生成超出人类经验的策略。

这种超越可能仍有边界，但当前尚看不到明确上限。总体上，我认为模型可以在已有知识基础上进一步推导和创新。实现路径不会只有一种，真实数据、合成数据、自博弈和其他方法都可能发挥作用。

## 4. 中国与美国的差距应如何缩小？

**答：** 过去的一种典型叙事是，中国模型落后美国约一年到两年，但只使用其二十分之一左右的算力完成相近工作。未来希望改写这一叙事：使用其几分之一的算力，同时把时间差缩短到六个月甚至三个月。

在总体算力仍有数量级差距的情况下，全面超越并不现实；但通过重点投入和取舍，在部分方向实现超越是可能的。

## 5. 持续学习的技术难点及其与 AGI 的关系

**问：** 持续学习或递归改进的最大技术难点是什么？为什么应当先解决持续学习，再进入通用智能？

**答：** 最大难点是目前还没有找到真正有效、可规模化的方法。全世界都处于探索阶段。我们有很多思路，也有一些看起来有前景的方向，但尚未完全做通。

公司内部比较重视一种研发闭环：下一版模型首先应当帮助我们自己提高研发效率。模型的第一目标不只是让外部用户觉得好用，而是要让 DeepSeek 自己在开发下一版模型时更高效。

如果模型能够有效帮助我们进行研究和工程开发，实现 AGI 的速度就会加快。当前 AI 仍然主要与人协作，尚不能完全自主工作，但已经能够明显提高研发效率。

持续学习之所以关键，是因为当前 Agent 的能力上限很大程度上受制于不能长期积累经验。若先解决持续学习，AI 的研究能力会显著增强，并进一步加快通用智能研发。相反，如果完全依赖人工推进后续通用智能，研发过程会更加数据密集、人力密集，效率也更低。

## 6. 国产硬件与未来组织结构

**问：** 国产硬件需要多长时间追上？随着公司规模扩大，组织结构将如何调整？

**答：** 国产硬件可能还需要数年。首先要解决生态信心和软件适配问题，然后解决产能问题。未来一至三年，产能仍可能是主要约束；但我不认为五年之后仍然会长期卡在同一个产能问题上。

公司过去的组织结构非常分散，甚至可以说没有明确组织架构。随着人员规模扩大，必须进行调整。未来部分部门需要建立更严谨的层级结构，另一些研究部门仍可保持松散、扁平的形式。相关调整已经开始，因为在现有规模下，没有适当组织结构会影响项目推进。

## 7. 版本节奏与模型规模

**答：** 对我而言，较舒适的发版节奏大约是两到三个月一版。当前线上版本仍比较粗糙，很多能力需要继续改进；在没有意外的情况下，后续版本应当逐步提升。

在约 50B 激活参数规模上，我们预计与同规模领先开源模型的差距不会特别大，尤其在推理速度和效果方面。但与未公开的超大模型相比，仍可能存在显著差距。这种差距可能需要 150B 乃至更大激活参数规模才能缩小。

原文提及 150B—250B 激活参数以及若干对标模型名称，但 ASR 对模型代号的识别不稳定，无法可靠还原，因此不作进一步改写。

## 8. AGI 是连续过程还是临界跃迁？

**问：** AGI 的实现是渐进过程，是否意味着不存在明确临界点？

**答：** 它可能没有清晰临界点，但演进过程不是线性的。AI 可以加速 AI 研究，因此越往后，技术进步速度越可能呈现非线性。

目前没有看到语言模型 Scaling 的明确上限。无论以国内还是美国当前研究水平看，都尚未触及可确认的终点。

具身智能最终仍然需要进入技术路线。人的真实需求并不局限于计算机操作，而是衣食住行和现实劳动。如果目标是解决人力需求，具身智能无法绕开。

在没有具身能力之前，我们对 AGI 的一个重要要求，是它能够帮助迭代下一版模型；有了具身能力之后，也希望它能够帮助设计和迭代下一代机器人。

## 9. AI 是否缺乏“品位”和直觉？

**问：** 如果 AI 能够自我进化，研究中的品位、直觉和方向选择是否仍然重要？

**答：** 我认为 AI 当前并不明显缺乏品位和直觉，真正缺少的是持续学习能力。至少在写作等任务中，模型已经能够表现出相当的判断和风格能力。

## 10. 确定性工程目标与开放研究问题如何分配资源？

**问：** 对于可通过 Scaling 推进的确定性目标，与持续学习这类尚未解决的研究问题，应如何配置研究人员和算力？

**答：** 某些模型能力目标相对确定，但另一些目标仍只能视为方向。开放研究问题未必需要大量算力或固定项目团队。很多探索只需要少量计算资源，核心是想法。

我们把这种研究称为“摸奖”：进入门槛不高，很多人都可以思考和尝试，但最终谁能找到有效方法，并不容易预测。这类问题不一定需要传统意义上的集中资源分配，关键是公司愿意把它视为重要问题，持续讨论并允许多人探索。

### 幻觉问题

模型幻觉可以通过更好的后训练等方法持续改善。它是可解决、可缓解的问题，但需要长期投入。对我们而言，幻觉更接近产品和可靠性问题，必须处理，但不是当前最核心的基础研究瓶颈。

### 数据标注

高质量数据标注成本很高，中国在高端数据标注上未必具有明显成本优势。以国内资本投入结构，很难复制美国公司同等规模的高质量标注投入。

因此，需要区分不同类型数据：成本较低的数据可以优先标注，成本高的数据则需要更谨慎投入。当前大量核心研究人员实际上也在参与数据相关工作，因为在现阶段，数据仍然是解决模型能力问题的关键组成部分。

## 11. 中国公司的结构性优势

**问：** 除效率之外，中国模型在哪些方面可能优于美国模型？

**答：** 中国公司可能在成本和产品体验上形成结构性优势。

成本优势比较容易理解。美国公司资源更充足，未必把极致成本优化作为最高优先级；中国团队在资源受限环境中，会把效率视为核心能力。

产品方面，中国互联网公司长期积累了较强产品和运营能力，因此用户体验未必弱于美国。除此之外，暂时看不到非常明确的其他结构性优势。

## 12. 融资后是否会加大后训练投入？

**问：** Anthropic 和 OpenAI 在后训练、高质量数据等方面投入巨大。融资后是否会显著增加相关投入？

**答：** 会加大投入，但高质量数据的核心瓶颈不完全是资本，而是时间和组织能力。OpenAI、Anthropic 启动得更早，资本和算力也更多；国内在这一方向大规模投入的时间相对较短。

即使资本足够，团队扩张、流程建设和高质量标注速度也存在上限。当前相关体系正在快速扩张。大约一年内，国内高质量数据能力出现明显改善，是可以预期的，但它确实需要时间。

## 13. 是否进入金融、法律、医疗等垂直 Agent？

**问：** 未来是否会开发金融、法律、医疗等垂直应用？

**答：** 目前还难以判断国内最顺利的商业路径。国内市场结构与国外不同，现在还没有到可以确定长期模式的阶段。

以当前技术条件看，更合理的优先级可能是先做好通用 Agent，尤其是 Coding Agent。Coding Agent 能覆盖大量通用工作，并能够反过来提高模型研发效率。金融、医疗等垂直 Agent 的优先级现阶段相对更低。

## 14. 科研愿景与资本市场如何平衡？

**问：** 公司如何在纯粹科研、AGI 目标和资本市场责任之间取得平衡？

**答：** 我认为二者可以兼容。如果 B 端收入达到数亿美元，同时 C 端保持稳定用户基础，公司就已经具备商业基础。随着需求增长，公司可能逐步接近现金流和净利润转正，而不再处于纯烧钱阶段。

即使技术后续暂时没有重大突破，API 服务本身也可能支撑一家具有稳定业务的公司。最坏情况下，可以集中把 API 和相关服务做好。我们希望追求更大的目标，但同时也存在可支撑公司的商业底线。

## 15. 组织模式是否有历史参照？

**问：** DeepSeek 的组织形式是否存在可模仿的历史对象，例如贝尔实验室？

**答：** 我们没有直接模仿对象。每一步都是基于现实条件、实际利弊和当时的最优判断作出选择，而不是因为某个历史组织曾经这样做。

我们与贝尔实验室并不相同。贝尔实验室可以在特定制度条件下不直接承担商业化压力，而我们明确需要商业化并保持公司持续生存。我们可以有利润之外的追求，但本质上仍然是一家公司，不会获得政府长期无条件资助。

因此，B 端业务长期一定重要，因为公司可能需要依靠它维持运营；只是它在当前阶段不是最高优先级。历史上许多伟大公司都具有利润之外的追求，这种追求未必妨碍商业化，反而可能使商业化更成功。关键在于明确赚什么钱、何时赚钱、赚多少以及主动放弃什么。

## 16. TileLang 是否会降低推理效率？

**问：** 从 CUDA/PTX 生态转向 TileLang 等高级编程方式，是否会降低推理效率？

**答：** 总体上会提高开发效率，而不是降低。过去高度依赖 CUDA 生态，现在可以使用更简单的高级语言重写算子，代码量更少，开发速度更快。

目前 TileLang 代码仍主要由人编写，但已经比直接编写 CUDA 更高效。公司也在研究使用 AI 自动编写 TileLang。

在底层执行效率上，可能存在约 1%—2% 的损失，但这是可以接受的。相对于开发效率、跨硬件迁移能力和生态独立性，这种损失很小。
